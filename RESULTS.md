# 一般化スター高さ問題: 候補の分析・探索・構成の記録

作業日: 2026-07-22。本文書は、一般化スター高さ問題（generalized star height problem）
に関する一連の調査・計算機探索・明示的構成とその機械検証の記録である。

## 1. 問題の背景

- 一般化正規表現 = 正規表現 + 補集合演算子。言語の一般化スター高さ = 必要な
  Kleene スターの最小入れ子深さ。
- **未解決問題（1960年代から）**: 高さ 2 以上を必要とする正規言語は存在するか。
  高さ 1 以下かの判定アルゴリズムすら知られていない。
- 高さ 0（スターフリー）は Schützenberger の定理で特徴づけ済み
  （構文モノイドが非周期的 ⟺ 高さ 0、判定可能）。
- 既知の肯定的結果（高さ ≤ 1 が証明済みのクラス）:
  - 可換群が認識する言語（Henneman）
  - 冪零類 2 の群、すなわち L(u,k,n)（|u| ≤ 2）（Pin–Straubing–Thérien 1992）
  - L(u,k,n) で |u| ≤ 3 かつ n が平方因子なし（PST 1992）
  - 可換群 ⋊ (Z/2)^n の divisor、wreath M∘(G∘N)（M,N 非周期的、G 可換）の
    divisor（PST 1992）
  - 位数 12 未満の群が認識するすべての言語（PST 1992）
  - セグメント counting 言語 W(h,k,r,m)（Thomas 1981）

## 2. 構造的障害: なぜ下界の不変量が存在しないか

- **Perrin (1975)**: 有限双接頭符号（finite maximal bifix code）の群は、
  次数 n の推移的置換群で n-サイクルを含む。次数 5 では A5 や S5 が実現される。
- 有限符号 X はスターフリーなので X* は高さ ≤ 1。その構文モノイドの最小イデアルに
  A5 が座る。⟹ **高さ 1 の言語の構文モノイドは非可解群を含み得る**。
- 帰結: 群の割り切り（division）に基づく不変量（可解性、擬恒等式、variety 論）は
  高さ 1 と 2 を分離できない。division に安定でない道具（作用の座標を見る道具）が
  必要。候補: Margolis–Rhodes–Schilling の flow 理論（2024、KR 複雑性判定可能性）、
  Place–Zeitoun の star-free closure に対する covering 判定（2019–2023）、
  Gehrke–Grigorieff–Pin 双対性による固定アルファベット上の射有限語方程式。

## 3. 候補言語の特定

当初、既知の結果をすり抜ける最小の候補として以下の 2 つを特定した。

- **候補 1**: L(aab,0,4) = { w ∈ {a,b}* : (aab の散在部分語としての出現回数)
  ≡ 0 mod 4 }。|u| = 3 かつ法 4（平方因子あり）は PST が明示的に未解決とした
  パラメータ領域。
- **候補 2**: A4 の word problem。φ(a) = (123), φ(b) = (12)(34) として
  L = { w : φ(w) = e }。位数 12 未満は PST により高さ ≤ 1 で、位数 12 の群のうち
  A4 以外（C12, C6×C2, Dih6, Dic3）は既存定理で処理できる。最初の 2 群は可換、
  Dih6 = C6 ⋊ C2 は PST の半直積定理の直接例である。Dic3 についても、下記の
  明示的埋め込みにより同じ PST 定理に帰着する。したがって、Bourne (2017) が
  A4 と Dic3 を未解決として挙げた位数 12 の表は、Dic3 についてはこの別経路で
  補える。A4 はこの PST クラスを割らないことを、可換正規部分群との交わりから
  得る完全列と A4 の正規部分群 1, V4, A4 の列挙により確認した。

### 位数 12 の群の監査と Dic3 の帰着

位数 12 の群は同型を除いて C12, C6×C2, Dih6, A4, Dic3 の 5 群である
（Sylow 論法による標準的分類。例えば Dummit–Foote, *Abstract Algebra*
3rd ed., §5.5 の演習水準）。
PST (1992) は「可換群 A と基本可換 2 群 E = (C2)^n の半直積 A ⋊ E を
割る群が認識するすべての言語は一般化スター高さ ≤ 1」と証明している
（論文 abstract および §7、位数 < 12 の帰結は Corollary 7.7）。

なお A4 はこのクラスを割らない。実際、S ≤ A⋊E と N ◁ S に対して
Q = S/N と置くと、B = (S∩A)N/N は Q の可換正規部分群であり、
Q/B は E の部分群の商、従って基本可換 2 群である。Q ≅ A4 なら
B は 1 または V4 だが、対応する Q/B はそれぞれ A4 または C3 となり、
どちらも基本可換 2 群ではない。

Dic3 = ⟨a,b | a³ = b⁴ = 1, b⁻¹ab = a⁻¹⟩ に対し、

    H = (C3 × C4) ⋊ C2,
    τ·(x,y) = (−x,y)

と置く。H の元を ((x,y),ε) と書き、

    a ↦ ((1,0),0),    b ↦ ((0,1),1)

とすると a³ = b⁴ = 1 と b⁻¹ab = a⁻¹ を満たす。さらに
定義関係により任意の語は aⁱbʲ（0 ≤ i < 3, 0 ≤ j < 4）に簡約できる一方、
これら 12 個の像は H の座標からすべて相異なる。従って提示群はちょうど
12 元であり、これは Dic3 の H への単射 ι を与える。なおこの提示
⟨a,b | a³ = b⁴ = 1, b⁻¹ab = a⁻¹⟩ は、標準的 dicyclic 提示
⟨x,y | x⁶ = 1, y² = x³, y⁻¹xy = x⁻¹⟩ で x = a², y = b と置いたものと
同型であり、位数 12・非可換・位数 4 の元の存在から位数 12 の 5 群のうち
Dic3 と一意に同定できる（A4 の元の位数は 1,2,3、Dih6 は 1,2,3,6）。
H は「可換群 C3×C4 by C2」という PST のクラスそのものである。
Dic3 が認識する任意の言語 L = φ⁻¹(P) は、受理集合を ι(P) と取れば
ι の単射性より (ι∘φ)⁻¹(ι(P)) = φ⁻¹(ι⁻¹(ι(P))) = φ⁻¹(P) = L となり
H にも認識されるため、一般化スター高さ ≤ 1
である。これは A4 の §5–§5.5 の新しい counting 構成を必要としない。

**機械検証（2026-07-23）**: 上の埋め込みは `scripts/dic3_embedding.py` で
全数検証した — 関係式 a³ = b⁴ = 1, b⁻¹ab = a⁻¹、12 個の像 aⁱbʲ の相異、
非可換性と位数プロファイル {1:1, 2:1, 3:2, 4:6, 6:2}（位数 12 の 5 群の
中で Dic3 を一意に同定する）、A = C3×C4 が H の可換正規部分群で
|H/A| = 2 であること、および H の位数 12 部分群（3 個）のいずれも
A4 の位数プロファイル {1:1, 2:3, 3:8} を持たないこと（非分割論法の
有界サニティ版）。全チェック PASS。

従って位数 ≤ 12 のうち、文献と上の群論的帰着だけで解決するのは A4 以外の
全群である。A4 については、2026-07-23 の更新で還元側の状態が変わった:
全元アルファベット還元（notes/conway_group_identities_and_full_alphabet.md
命題 3.1）が使う二つの閉性（一文字左商・letter-to-letter 逆準同型）に
同ノート §3.5 で自己完結の帰納法証明を与え、敵対的査読 1 回
（2026-07-23、エッジケース含め破綻なし）と機械コンパニオン
`scripts/closure_lemmas_check.py`（400 式・厳密 DFA 同値・全 PASS）を
付したため、`FULL-ALPH-RED-01` は PST 仮説監査に依存しない `PROVED` と
なった。合成の結論「全 A4-認識言語の一般化スター高さ ≤ 1」
（`A4-ALLLANG-01`）は `COMPUTED` — 上限は基底 `A4-FULL-01` の水準で
決まり、`PROVED` には昇格しない（弱い環は全元版の再構成チェックが
有界長全数＋ランダムである点）。独立な人間査読は全行で未了。

**まとめ（位数 ≤ 12）**: 「位数 ≤ 12 の任意の群 G が認識する任意の言語は
一般化スター高さ ≤ 1」が、次の検証水準で本リポジトリの記録になった —
位数 < 12: `CITED`（PST Corollary 7.7）。C12, C6×C2: `CITED`（可換、
PST）。Dih6 = C6⋊C2: `CITED`（PST 半直積定理の直接例）。Dic3: `CITED` +
埋め込みの群論計算は `PROVED`／機械検証済み（`scripts/dic3_embedding.py`）。
A4: `COMPUTED`（§5–5.5）+ 還元 `PROVED`。

**Bourne (2017) との関係**: Bourne の梯子は位数 12 で止まり、A4 と Dic3 が
未解決として残された（台帳 `SMALL-12-01`）。その障害は、引用系
A ⋊ C2 論法の C3 への拡張が一意分解ステップで破れること
（`SMALL-C3-FAIL`）にあった。本節の Dic3 埋め込みは C3 を**可換部分
C3×C4 の内部**に入れ、作用群を C2 のまま保つことで、この C3 障害を
迂回する（Bourne の議論の修理ではなく、PST の既存クラスへの帰着）。
A4 側は §5–5.5 の counting 構成＋命題 3.1 で落ちる。従って Bourne の
位数 12 障壁は、上記の検証水準の限定つきで両側とも通過した。

### 候補 1 の算術的分解

m_j = j 番目の b の前の a の総数とすると binom(w,aab) = Σ_j binom(m_j,2)、
binom(m,2) mod 4 は m mod 8 で決まる。m のビットを b2 b1 b0 とすると

    binom(m,2) ≡ b1 + 2(b2 + b0·b1)  (mod 4)      [全 8 剰余で検算済み]

よって N_r^{(m)} = #{ b : a-カウント ≡ r mod m } とおくと

    L(aab,0,4) は M1 = N_2^{(4)} + N_3^{(4)} mod 4 と
                 M2 = N_3^{(8)} + N_4^{(8)} + N_5^{(8)} + N_6^{(8)} mod 2
    の Boolean 結合。

### 候補 2 の代数的分解

A4 = V4 ⋊ Z/3、t = (123) は V4 の非自明元 x → z → y を巡回置換する。
t を右へ押し出す標準形 φ(w) = c(m_1)···c(m_N)·t^{|w|_a}（c(m) = t^m x t^{-m}）より

    w ∈ L_{A4} ⟺ |w|_a ≡ 0 (3), N_0+N_2 ≡ 0 (2), N_1+N_2 ≡ 0 (2)
    （N_r = N_r^{(3)}、段階 3 の累積カウント）

つまり両候補とも「**単一剰余の段階カウント言語** W(h,k,r,m) =
{ w : N_r^{(m)} ≡ h mod k } の Boolean 結合」に帰着する。

## 4. 計算機探索（search.py / verify.py）

- 部品: LC = 文字数剰余カウント、TOK(X,c) = 有限トークン集合 X（トークンは
  a, b, B=b* 上の列 = スターフリー）の X*-分解トークン数 mod c の決定化状態。
  各状態指示言語は高さ ≤ 1。
- 判定: ターゲット値が特徴量状態の関数か。サンプル照合（フィルタ）＋
  積オートマトン全状態探索（通れば厳密な証明）。
- 結果:
  - 対照実験: Thomas 型言語（偶数個の a の後の b の個数 mod 2）は
    LC + TOK({b, ab*a}, 4) の Boolean 結合であることを**積オートマトンで証明**
    （既知定理の機械的再発見。パイプラインの妥当性確認）。
  - トークン集合 6120 通りの掃引: PST 型・M1・M2・FULL のサンプル生存者ゼロ。
    L ≤ 12 で生き残った特徴量ペア 3 組も長さ ≤ 200 のランダム 30 万語で反例。
  - **較正**: 解決済みの PST 言語（binom(w,aab) mod 2）すらこの設計空間で
    表現不可能 ⟹ 探索の失敗は「高さ 2 の証拠」ではない。
    連接（オープナー等）を含む式空間が必要という教訓。

## 5. 設計による構成と機械検証（a4_attempt.py 等）

### 構成（単一剰余 W(h,k,r,m) の高さ 1 表現）

- オープナー O_r = (b*a)^r （スターフリー、位相 r に到達）
- トークン X = { b, a(b*a)^{m-1} } （コネクタは a を m 個消費、内部ブロックは
  位相 r+1..r+m-1 = すべて非マーク）
- X は接頭符号（分解が一意）、#tok = N_r + (消費した a の数)/m
- 尾部 ρ ∈ { ε, ab*, ab*ab*, ... }（m-1 種）で任意の語を一意に階層化
- N_r mod k は「#tok mod k（X^j(X^k)* で判定）」と「|w|_a mod mk」から復元

スターの入れ子は (XX)* 等の 1 段のみ（b* はスターフリー）なので高さ ≤ 1。

### 検証結果（3 層）

1. **原子部品の厳密証明**（積オートマトン全状態探索、サンプリングでない）:
   - W(stage 3, r∈{0,1,2}, mod 2)、W(stage 4, r∈{2,3}, mod 4)、
     W(stage 8, r∈{3,4,5,6}, mod 2) すべて「値 = 高さ1特徴量の関数」を証明。
   - 負のコントロール（誤ったオープナー）は正しく失敗。
2. **A4 の独立 end-to-end 検証**（a4_final.py): 式スキーマを Python `re`
   エンジン（探索コードと独立の実装）で評価し、置換の直接合成と比較。
   長さ ≤ 16 の全語（131,071 語）＋長さ ≤ 120 のランダム 3 万語で完全一致。
3. **候補 1**: 証明済み W 原子の値から算術的に決まるため高さ ≤ 1。
   合成もランダム 15 万語＋全数 L ≤ 13 で確認。

### 明示的な式（N_0^{(3)} 偶の例、independent_check.py で独立検証済み）

    X  = b | a b* a b* a          （スターフリー）
    D  = (XX)* ,  XD = X(XX)*     （唯一のスター段）
    A_q = { w : |w|_a ≡ q mod 6 } （高さ 1）

    N_0 偶 ⟺ w ∈ (D∩A_0) ∪ (XD∩A_3) ∪ (D·ab*∩A_1) ∪ (XD·ab*∩A_4)
                ∪ (D·ab*ab*∩A_2) ∪ (XD·ab*ab*∩A_5)

長さ ≤ 18 の全語（524,287 語）＋ランダム 20 万語（長さ ≤ 300）で
`re.fullmatch` により検証、完全一致。

## 5.5 全要素アルファベット（multi-mover）の A4（a4_full.py / a4_full2.py / a4_full3.py / a4_full12.py）

「A4 の全 12 元をアルファベットとする word problem」は、複数の文字が
Z/3 位相を動かす（multi-mover）ため §5 の分業構成が直接は使えない。
3 段階の試行を経て**高さ 1 であることを機械検証つきで解決**した。

### 障害と 2 つの鍵

- 第 1 試行（a4_full.py、存在的パーズ＋マーク）は、トークンのエントリー位相が
  2 種類になり「型付き隣接」が生じて自由スター 1 段で書けず失敗。
- **鍵 1: パターン条件つきカット。** カットを常に「位相が q になる瞬間」に固定し、
  「直前の文字列がパターン π で終わる場合だけ例外的にトークンを継続」とする。
  これで全トークンのエントリー位相が q に統一され、式は O·X*·(接頭辞) の形になる。
  ε の算術より、ムーバー終端パターンはカット境界をまたげない（パターン尾部は
  q から離れる方向に歩く）ので、トークン言語 X が文脈独立に定義でき、
  遷移モノイドの非周期性で星無し性を機械認証できる。
  認証結果（{u,d,k} 版）: 全カット、非ムーバー除外、hg 型（h≠g）はすべて
  非周期的 = OK。**同一文字の純粋冪パターン（uuu 等）だけが非周期性を破る**
  （u^n が位相を自由巡回するため）。このため前向きだけでは
  N[u,p], N[d,p] の一部が決定できない（a4_full2.py で厳密に確認）。
- **鍵 2: 反転（reversal）。** 高さ 1 は語の反転で閉じる。文字 g の
  前向きエントリー位相 p と後ろ向きエントリー位相 r は r = P − ε_g − p
  （P は総位相）で結ばれ、**ε_g に依存してシフトが異なる**。この非対称性により
  前向きの抽出量（ムーバー到着和 S_q = N[u,q−1] + N[d,q−2]）と
  後ろ向きの抽出量（同位相和 C_s = N[u,s] + N[d,s]）が独立な一次結合を与え、
  文字数 mod 2 と合わせて GF(2) 上の連立系がフルランクになる。

### {u,d,k} 版（ε = 1, 2, 0）の解

明示式（すべて mod 2、Z_q = 全カット特徴、z_q = N[k,q]、チルダは反転語上）:

    z_q = Z_q + F(q,'k'),   S_q = Z_q + z_q,   C_s = S̃_{P−s}
    N[u,0] = |w|_u + S_0 + C_1,  N[u,1] = |w|_u + S_1 + C_2,
    N[u,2] = |w|_u + S_2 + C_0,  N[d,p] = C_p + N[u,p]

a4_full3.py: 長さ ≤ 12 の全 797,161 語＋ランダム 2 万語（長さ ≤ 400）で
恒等式を検証、完全一致。さらに「A4 の到達元は (P, N[g,p] mod 2) の関数」を
積オートマトン（384 状態）の全探索で**厳密に証明**。

### 全 12 元アルファベット版

ムーバー 8 文字については同一文字ペア gg のパターンカウント n_q = N'[gg,q] が
未知数として残るが、前向き方程式は x_p + n_{p+ε}、後ろ向き方程式は
x_m + n_{m+2ε} の形になり（gg の反転は gg だが最終文字が入れ替わるため
位相シフトが ε から 2ε に変わる）、ε ≠ 0 では ε ≢ 2ε (mod 3) なので
2 系統＋文字数和で 6 未知数がちょうど解ける。a4_full12.py:

- 93 個のトークン言語すべての非周期性を認証（失敗 0）
- 特徴量とイベントカウントの一致（境界非跨ぎ性）: 2400 ケース一致
- 全 N[g,p] の復元: 長さ ≤ 4 全 22,621 語＋ランダム 4000 語（長さ ≤ 300）一致
- 群要素 = f(P, N[g,p]) の合成公式: ランダム 2 万語一致

**結論: A4 の全 12 元をアルファベットとする word problem も一般化スター高さ 1。**
生成元の取り方によらず、A4 は反例候補から完全に外れた。

## 5.6 A5 の生成系依存の高さ 1 表現（A5_generator_dependent_star_height_1.md / a5_check.py）

生成系を選べば **A5 の word problem すら高さ 1 になる**。
π(a) = (123), π(b) = (145) とすると、a は {4,5} を固定し b は {2,3} を
固定するため、点 1 以外では「動かす文字」が一意で、点 1 への初回帰還語は
強制歩行になる:

    C = a b* a b* a ∪ b a* b a* b   （b* = ¬(Σ*aΣ*) 等により星無し）

点 i の固定言語 K_i（i = 1,2,3）はそれぞれ

    K_1 = C*,   K_2 = b* ∪ b* a b* a C* a b*,   K_3 = b* ∪ b* a C* a b* a b*

と書け、A5 で 3 点 1,2,3 を固定する元は単位元のみ（(45) は奇置換）なので

    L_e = π^{-1}(1) = K_1 ∩ K_2 ∩ K_3

は高さ ≤ 1。全射性より構文モノイドは A5（非周期的でない）なので、
**高さはちょうど 1**。商を取れば全ファイバー π^{-1}(g) も高さ 1。

- 機械検証（a5_check.py）: ⟨(123),(145)⟩ = A5（位数 60）、3 点固定元の
  自明性、および式と直接計算の一致を長さ ≤ 16 の全 131,071 語＋
  ランダム 20 万語（長さ ≤ 400）で確認、完全一致。
- 副産物として **K_1 = C* 単独でも構文モノイドは A5**（Stab(1) ≅ A4 の
  core が A5 単純性より自明のため）。つまり「構文モノイドが非可解単純群
  A5 そのものである高さ 1 の言語」の明示例になっており、Perrin の有限
  双接頭符号（群が構文モノイドに部分群として現れるだけ）より強い形で
  「群の複雑さ単独では下界にならない」ことを示す。
- 機構は A4 の分業生成元と同一（「各文字が相手の台を固定する」）。
  生成系の台がほぼ交わらないよう選べたことが本質で、ATLAS 型
  (2,3,5)-生成元や全 60 元アルファベットには波及しない
  （文字→長い語の逆準同型は長さ保存でなく、高さ保存が未知）。
- 帰結: word problem の高さは生成系の取り方に強く依存し、
  「A5 だから高さ ≥ 2」という形の候補設定は生成系を指定しない限り無意味。
  残る候補は **全 60 元アルファベットの A5**（あらゆる生成系を支配する
  普遍ケース）と、台の分離が不可能な生成系（例: (12)(34), (135) など
  (2,3,5)-型）に絞られる。

## 5.7 A5 全要素アルファベットへの拡張の試み: 障害の特定（a5_frontier.py）

§5.6 の構成を一般基準として定式化した:

**組み立て補題（機械判定可能な十分条件）.** 生成集合 S が A5 を生成し、
あるアンカー点 p について「p を出口 E に置き換えた内部歩行機械
（START, 内部 4 点, E, DEAD）の遷移モノイドが非周期的」ならば、
初回帰還符号 C（START→E）、開部品 B_i（i→E）、閉部品 D_i（START→i）、
回避部品 A_i（i→i, p 非訪問）はすべて星無しであり、

    K_p = C*,   K_i = A_i ∪ B_i C* D_i,   L_e = K_{i1} ∩ K_{i2} ∩ K_{i3}

（3 点の固定化群は自明）により word problem は高さ ≤ 1。

### 判定結果（a5_frontier.py）

| 生成系 | 通過アンカー | 帰結 |
|---|---|---|
| (123),(145) | 1 | 高さ 1（§5.6 の再導出） |
| (12345),(123) | 1,2,3 | **高さ 1（新規肯定例）** |
| (123),(345) | 3 | **高さ 1（新規肯定例）** |
| (12)(34),(135)（(2,3,5) 型） | なし | 本手法は不適用 |
| 全 60 元アルファベット | なし | 本手法は不適用 |

パターンは明確で、**各生成元が「アンカーを含む単一サイクル＋固定点」である
こと**が通過の実質条件になる（アンカー上のサイクルは出口で破壊され、
固定点は自己ループで無害）。二重互換 (12)(34) は 2 つのサイクルを持ち、
1 つのアンカーでは片方しか破れないため、(2,3,5) 型生成系は全アンカーで
失敗する。3-サイクル同士は 5 点上で必ず点を共有する（3+3 > 5）ので、
**2 つの 3-/5-サイクルからなる生成系はすべて高さ 1** に落ちる。
3 個以上の生成元では全サイクルの共通点が必要（§5.8 の
{(123),(145),(245)} は共通点がなく失敗する）。

### 全要素アルファベットが現手法の全ルートを破ること

1. **アンカー基準**: どの点 p でも Stab(p) ≅ A4 の 11 個の非自明文字が
   内部 4 点を出口なしで置換し続け、内部モノイドは A4 を含み非周期的
   でない（機械確認済み）。パターン例外つきカットは 1-訪問時の挙動しか
   変えられず、内部巡回を殺せない。
2. **正則作用（60 点）**: 文字 g の作用は不動点なし。x ∉ ⟨g⟩ の剰余類
   x⟨g⟩ は単位元を経由せずに g で巡回するため、単位元への初回帰還符号は
   星無しでない。
3. **商位相の不在**: 5 点作用は原始的（非自明合同 0 個、機械確認済み）、
   A5 は単純（アーベル化自明）なので、§5.5 で A4 に使った「巡回商への
   位相準同型＋カット特徴＋反転」の枠組みには取り付く点が存在しない。
   反転トリックは認証済み特徴が 1 つもない状況では何も生まない。
4. 複数アンカーのカットはエントリー点が複数になり、型付き隣接
   （§5.5 で特定した catch-22）に戻る。

**結論: 全 60 元アルファベットの A5 word problem は現有のすべての手法で
高さ 1 を示せず、障害の位置が正確に特定された形で未解決に残る。**
最小の未解決インスタンスは (2,3,5) 型生成系 {(12)(34),(135)}
（障害は二重互換の破れない第 2 サイクル、周期はわずか Z/2）と、
全 60 元アルファベット（障害は Stab(p) ≅ A4 全体）である。前者は
障害群が可解なので、A4 で開発した可換カウンタ抽出を歩行構造の上に
重ねる「2 層構成」が次の攻略候補になる。

## 5.8 (2,3,5) 型生成系の解析: 2 つの不可能性定理（a5_235.py）

s = (12)(34), t = (135) に対する 2 層構成を試み、失敗の原因を
定理の形で確定した。

### 構造の精密化

5 点は列 α = {1,2}, β = {3,4}, γ = {5} に分かれ、s は列内でレベルを
反転（γ では固定）、t は上段の点でだけ列を α→β→γ→α と進める。
歩行状態は相互依存カウンタ対

    E = #(レベル偶のときの t) mod 3   … 列
    G = #(列 ≠ γ のときの s) mod 2    … レベル

で書ける。A4 で機械が機能した根拠は「位相 = 文字数準同型」という
**可換な接地**だったが、A5 は完全群（アーベル化が自明）なので E, G を
接地する準同型が存在せず、互いを条件にした循環参照のまま残る。
E-イベントでカットすればトークン内で s が自由に往復し（Z/2 巡回）、
G-イベントでカットすれば t が自由に巡回し（Z/3）、両方でカットすれば
エントリー状態が複数になり型付き隣接（§5.5 の catch-22）に戻る。

### 不可能性定理 1（全作用アンカー障害、機械確認済み）

A5 の非自明な推移的作用は共役を除いて 8 個（次数 5, 6, 10, 12, 15,
20, 30, 60）。そのすべてについて s, t のサイクル型を計算した結果、
**どの作用のどの点も「全生成元の全非自明サイクルに乗る」ことはない**
（例: 6 点作用で s は 2+2、t は 3+3）。よってアンカー法（§5.7 基準）は
作用の取り替えでは救えない。理由は単純で、忠実な作用では位数 2 の元は
必ず 2 個以上の 2-サイクルを持つ（1 個なら奇置換）ため、1 点では
破れない。**対合を含む生成系はアンカー法の適用範囲外**である。
さらに対合なしでも {(123),(145),(245)}（3-サイクルだが共通点なし）は
全作用・全アンカーで失敗する。対照実験として {(123),(145)} は
自然作用のみ・共通点のみで通過（§5.6 と整合）。

### 不可能性定理 2（可換カウント法の除外、割り切り論法）

認証済みカット特徴・文字数・反転特徴の言語の構文モノイドは
（非周期トークン層＋mod-k カウント層の積み上げなので）**可解な部分群
しか含まない**。Boolean 結合はモノイドの直積で認識されるが、単純群が
直積を割るならある因子を割る。A5 は非可解単純なので、
**A4 で開発した機械（パターン付きカット＋反転＋文字数、およびその
任意の有界拡張）の Boolean 結合では、構文モノイドが A5 の言語は
原理的に表せない**。§5.5 の成功は V4 ⋊ Z/3 の可解性そのものだった。
非可解な内容は §5.6 の C* のような「星無し符号の自由反復」からしか
生じ得ず、その符号の既知の供給源（点アンカーの初回帰還）は定理 1 で
封じられている。

### 魅力的な袋小路: 導出生成元

st = (12345), st² = (12534) は**ともに 5-サイクル**で、導出アルファベット
{st, st²} 上の word problem はアンカー基準を全点で通過する（高さ 1）。
しかし元の語への引き戻しは u_c ↦ s t^{c+3k} という置換であり、
(ttt)* が星の内側に入って高さ 2 になる。さらに t-ブロック長 ≡ 0 mod 3
の語では裸の対合 s が再出現するため、この経路では解決しない。

### 帰結

(2,3,5) 型（および対合を含む任意の生成系、共通点なし 3-サイクル系）の
A5 word problem を高さ 1 にするには、**点アンカー由来でない星無し符号で
非可解群の内容を持つもの**という、現在知られていない新しい供給源が
必要である。これは反例候補としての地位を強める材料でもある:
高さ 1 の既知の全機構が構造的に尽きる最初の明示的インスタンスが

    L_e over {(12)(34), (135)}   および   {(123),(145),(245)}

である（もちろん下界を証明する道具は依然として存在しない）。

## 5.9 段階付き ba*b 対カウント(「Weis L2」族)ステージ 2 の解決(weis_l2_family.py)

§6.3(下記 4.)の「Weis の L2」に対応する言語族のうち、位相 mod 2 の場合を
**高さ 1 で完全解決**した。詳細は `notes/weis_l2_stage2_height1.md`。

- 対象: N_{p,q} = #{連続 b 対: 先頭 b の位相 p、後続 b の位相 q}(位相 =
  累積 a 数 mod 2)の mod 2 / mod 4 言語と、セグメント剰余カウント
  T'(奇数長 a-ラン数)mod 4 の Boolean 閉包。
- **鍵 1(§6.3 の障害の解消)**: 無限符号の代わりに**有限**接頭符号
  X = {b, aa, ab} のフラット全トークン計数を使う。奇数ランがちょうど
  1 個の ab トークンを生み、厳密な整数恒等式
  `#ab = |w|_a + 2|w|_b − 2·#tok − [尾部 = a]`（全語長 ≤ 16 で全数検証）
  により T' mod 4 が (|w|_a mod 4, |w|_b mod 2, #tok mod 2, 尾部) の関数になる。
  トークンが有限なので星無し性は自明で、(X²)* の 1 段の星だけで済む。
- **鍵 2**: 奇数ランの開始位相は厳密に交互(位相ウォークの反転構造)。
  これで対行列全体が T' mod 4・W 原子(stage 2, mod 4; §5 構成で同時検証)・
  文字数・境界旗に還元される。還元は設計指針にとどめず、
  **積オートマトン全探索で「特徴状態の関数」を厳密証明**([4])。
- 検証 3 層: 厳密証明(積オートマトン)+セル表 vs 直接カウント
  (ランダム 3 万語、27 万比較一致)+明示式の `re` 独立検証。
- 副産物: Thomas (1981) のセグメント定理の d = 2 の場合の自己完結な
  機械証明(CITED → COMPUTED)。
- **ステージ 3 は未解決**: 単一ラン被覆符号の整数性が線形結合を
  (2T₁ + T₂) mod 3 型に固定し、必要な (T₁ + T₂) 型は単独トークン a を
  要求して充填子 aaa と接頭衝突する。カスケード符号 Z₃ も積オートマトン
  判定で不十分。T₁ mod 2 のみ厳密に得られる。障害の正確な位置は
  ノート §5 と PROOF_OBLIGATIONS.md N-L2-M3-001 に登録。
- **一次資料監査（2026-07-23 更新、notes/weis_2011_primary_audit.md）**:
  Weis 2011 の全文を Wayback Machine 経由で取得し監査した。判明事項:
  (i) 実物は L2 = L((ab\*a ∪ ba\*b(ab\*a)\*ba\*b)\*) で、**提案者は
  Weis でなく PST 1992**（"proposed in [30]"）。
  (ii) **Weis はフル版 L2 の高さ 1 を証明していない**（p.115 で明示的に
  未解決。制限版のみ高さ 1）。2009 年講演要旨の「4 候補すべて高さ 1」は
  論文と不整合で、フル版への読み替えは誤り。
  (iii) 「位数 48 の群」は実在する: 印字どおりの正規表現から再計算して
  構文モノイド ≅ C₂×S₄（非冪零、48 元）を再現（`scripts/weis_l2_actual.py`）。
  従来の「48 は再現されず」は、二次情報から再構成した族が実物と
  異なっていたことによる誤記録であり、本行で訂正する。
  (iv) **本節の stage-2 族定理は実物のフル版 L2 を捕まえない（厳密判定、
  2026-07-23）**: 印字どおりの L2 の最小 DFA（6 状態）と認証済み特徴族
  {LC(8,4), Flags(2), CodeTok({b,aa,ab}) mod 4, W(2,0,4), W(2,1,4)} の
  積オートマトン全探索（到達可能 114 状態）で、membership は特徴状態の
  関数でない。最短反例: `bababbb` ∉ L2 と `bbababb` ∈ L2 は全特徴・
  全認証原子値（N_{p,q} mod 4、T' mod 4）が一致する。原子追加・法倍加・
  --m3 構成子（X3, Z3, W(3,r,6)）の拡張でも NOT a function（反例
  `baaababbb`/`bbabaaabb`）。`scripts/weis_l2_actual.py`。
  **従って WEIS-L2-M2-01 はフル版 L2 の高さ ≤ 1 を導かず、L2 の一般化
  スター高さは依然オープン**（高さ ≥ 2 の主張ではない — 研究ルール 1）。
  フル版 L2 は「PST 1992 が提案し、Weis 2011 が未解決のまま残し、
  本リポジトリの認証族の外にあることが機械確認された」候補として、
  (2,3,5) 型と並ぶ反例候補に昇格する。構文モノイド C₂×S₄ は**可解**なので、
  非可解群系の候補とは独立の軸を与える。

## 6. 結論

- **位数 ≤ 12 の任意の群が認識する任意の言語は一般化スター高さ ≤ 1
  （検証水準つき、§3「まとめ」）。** 位数 < 12 と C12・C6×C2・Dih6 は
  `CITED`（PST 1992）、Dic3 は §3 の明示的埋め込み
  `Dic3 ↪ (C3×C4)⋊C2`（`CITED` + 群論計算 `PROVED`、機械検証
  `scripts/dic3_embedding.py`）、A4 は §5–5.5（`COMPUTED`）+ 命題 3.1
  （閉性の自己完結証明により `PROVED`、notes §3.5）による
  `A4-ALLLANG-01`（`COMPUTED`）。**Bourne (2017) の A4/Dic3 障壁は
  この検証水準の限定つきで両側とも通過**（Dic3 は C3 障害の迂回、
  A4 は新しい counting 構成）。独立な人間査読は未了。
- **候補 2（A4 word problem、生成元 (123),(12)(34)）: 一般化スター高さ 1。
  反例ではない（`COMPUTED`）。**
- **A4 word problem は全 12 元アルファベットでも一般化スター高さ 1
  （§5.5、`COMPUTED`）。** 全 A4-認識言語への還元は `PROVED`（命題 3.1、
  §3.5 追記）だが、合成結論の水準は基底の `COMPUTED` を継承する。
- **A5 word problem も生成系 (123),(145) では一般化スター高さ 1（§5.6）。**
- **単一サイクル生成元が共通のアンカー点を持つ A5 の生成系はすべて
  高さ 1（§5.7）。全 60 元アルファベットは現手法の全ルートが破れ、
  未解決（§5.7）。**
- **(2,3,5) 型生成系は 2 つの不可能性定理により既知の全機構の外に
  あることが確定。最有力の反例候補（§5.8）。**
- **候補 1（L(aab,0,4)）: 一般化スター高さ 1。反例ではない。**
  PST が 1992 年に未解決としたパラメータ（|u|=3, n=4）のこの代表例は、
  単一剰余への分解＋繰り上がりビット分解で落ちる。
- **「Weis L2」型の段階付き ba*b 対カウントは、位相 mod 2 の全 Boolean
  結合で一般化スター高さ 1（§5.9）。** 有限ラン符号 {b, aa, ab} の
  フラット計数が §6.3 のトークン化障害を回避する。位相 mod 3 は
  障害が特定された形で未解決。

### 新規性についての留保

- Thomas (1981) の W 言語は**セグメント** a^i b の i（局所連長）のカウントであり、
  ここでの**累積**カウントとは異なる（Weis 2011 も明示的に区別している）。
- 一方 PST (1992) は「transfer lemma で Thomas の結果の簡単な証明が得られる」と
  述べており、累積版もその周辺で当時から知られていた可能性がかなり高い。
  その場合、累積カウント構成の新規性に関する当初の推論は誤りだったことになる。
  また Bourne の表で未解決とされた Dic3 は §3 の PST 埋め込み経路で処理できるため、
  現在このリポジトリで実質的に残る位数 12 の証拠上の課題は A4 の
  `COMPUTED` / `UNREVIEWED` 部分である。PST の位数 12 未満定理は variety レベル
  （その群が認識する**すべての**言語）の主張であり、特定の生成元での
  word problem とは別物である点に注意。
- 新規性を主張する前に Thomas 1981、PST 1992 の transfer lemma、
  Robson の諸論文の精読が必要。

### 構成の適用限界（= 更新された候補リスト）

構成は「可換正規部分群 A ⋊ 巡回商 Z/m ＋ 生成元の分業（1 文字が巡回商を生成し
残りが A に入る）」に依存する。これが破れる最小の対象が新しい候補になる:

1. **A5 の word problem、(2,3,5) 型生成系 {(12)(34),(135)} および
   共通点なし 3-サイクル系 {(123),(145),(245)}**: §5.8 の 2 つの
   不可能性定理により、全 8 作用でのアンカー法と可換カウント法の
   Boolean 結合が原理的に尽きる最初の明示的インスタンス。
   現時点で最有力の反例候補。
2. **A5 の word problem、全 60 元アルファベット**: §5.7–5.8 により
   同じ 2 定理が適用され（対合を含む）、これも既知手法の外。
   1. が落ちればこちらが最前線に戻る。
3. **可解だが分業生成元を持たない埋め込み**: S4 の word problem
   （商 S3 が非巡回）。なお A4 については §5.5 の全要素アルファベット版が
   任意の生成元の場合を包含する（文字→群元の逆アルファベット射は高さ 1 を
   保つため、(123),(124) 生成なども自動的に高さ 1）。
4. **フル版 L2 = L((ab\*a ∪ ba\*b(ab\*a)\*ba\*b)\*)（PST 1992 提案・
   Weis 2011 未解決・構文モノイド C₂×S₄、位数 48、非冪零・可解）**:
   → **更新（2026-07-23）**: 一次資料監査（notes/weis_2011_primary_audit.md）
   により、これが実物の L2 であり Weis 自身がフル版を未解決のまま残した
   ことを確認。§5.9 の位相 mod 2 の族は有限ラン符号で高さ 1 に解決済み
   （2026-07-22）だが、**フル版 L2 は認証済み特徴族（拡張込み）の関数で
   ないことが厳密判定で確定**（§5.9 監査項 (iv)、`scripts/weis_l2_actual.py`）。
   従ってフル版 L2 は文献裏付きの生きた反例候補であり、1.–2. の非可解
   （A5）系とは独立な**可解群**の軸を与える。残るのは位相 mod 3 以上の
   族の場合も同様。

## 6.1 `exploring-math` PR #2 との接続（外部成果監査、2026-07-24）

2026-07-23 に
[`hora-algebra/exploring-math` PR #2](https://github.com/hora-algebra/exploring-math/pull/2)

がマージされた。以下は同リポジトリの証明・計算証拠を

### 役割分担と論理的な橋

- `gsh_bootstrap` は、一般化スター高さ 1 崩壊予想の Lean challenge、
  主張台帳、候補 DFA、式証明書とサイズ順合成探索を管理する形式化・
  ワークショップ基盤である。
- `exploring-math` は、半群・行列スター・relative orbit による大域還元と
  大規模な exact audit を担う探索研究ログである。
- 現在の `gsh_bootstrap` には `CORE2` の定義・定理はなく、Lean 側は
  直接 `GeneralizedHeightOneConjecture` を open challenge としている。

PR #2 の前提になっている
[`ELEMENTARY_EXPANSION_TWO_STATE_EQUIVALENCE.md`](https://github.com/hora-algebra/exploring-math/blob/main/ELEMENTARY_EXPANSION_TWO_STATE_EQUIVALENCE.md)
は、有限アルファベットごとに次を証明したと報告している。

\[
 \text{すべての正則言語が }GH_1
 \quad\Longleftrightarrow\quad
 \forall A,B,C,D\in SF,\ (A\cup BD^*C)^*\in GH_1 .
\]

右辺は star-free 成分を持つ固定2状態行列のスター問題（CORE2）の
スカラー形である。従って CORE2 は単なる部分問題ではなく、
`GSH/Challenges/GeneralizedStarHeight.lean` の challenge と論理的に
同値な小型中間定理である。この大域同値自体は PR #2 より前の成果で、
PR #2 はその右辺に対する有限監査と候補 architecture の境界を追加した。

### PR #2 から移植できる結果

1. **有限4096族の完了。** PR が固定した singleton-word operand の
   CORE2 4096 tuple はすべて $GH_1$ である。最後の2 tuple
   $(01,10,10,01)$ と $(10,01,01,10)$ には
   [明示的な高さ1式](https://github.com/hora-algebra/exploring-math/blob/main/generalized-star-height/CORE2_FINAL_PAIR_EXPLICIT_GH1.md)
   が与えられ、complete DFA の対称差 BFS で語長 cutoff なしに同値性が
   確認されている。これは有限族定理であり、任意の star-free operands
   への外挿ではない。

2. **$GH_1$ 上で Krohn--Rhodes 複雑性は非有界。**
   [有限 binary prefix-code の構成](https://github.com/hora-algebra/exploring-math/blob/main/generalized-star-height/FINITE_CODE_TN_LIFT_PROBE.md)
   により、各 $n\geq3$ に対し有限 binary prefix code $C_n$ で
   $T_n\prec M(C_n^*)$ が成り立つ一方、$C_n^*$ は定義から $GH_1$ に入る。
   さらに binary finite-code stars の構文モノイドは division-universal
   である。従って、KR complexity の有界性や proper pseudovariety
   だけでは generalized star-height 1 を特徴づけられない。これは
   RESULTS.md §2 の「群の複雑さ単独では下界にならない」を大幅に強化する。

3. **`A_5` は候補と compatibility gate を区別する必要がある。**
   PR #2 の固定 order-5/order-5 tuple $A=S$, $B=C=D=\varnothing$ では
   target $S^*$ の構文モノイドは $A_5$ だが、target 自体は直接 $GH_1$ である。
   [complete raw stamp の監査](https://github.com/hora-algebra/exploring-math/blob/main/generalized-star-height/CORE2_CONCRETE_A5_RAW_NUQ.md)
   は $2723$ 元を閉じ、その全 fibres に固定 stamp 上の $GH_1$ 証明を与える。
   これは本リポジトリの $(2,3,5)$ 型 `a5_235` word problem を解決しない。
   したがって両リポジトリは矛盾せず、`A_5` は一方では生きた候補、
   他方では一様 CORE2 構成が通過すべき hard compatibility test である。

4. **relative-orbit route の正確な未解決境界。** 固定 `C4` benchmark では、
   33 proper derivatives から作る全 $2178$ one-marker pool を加えても
   period 4 witness が残る。一方 canonical full one-marker
   Schützenberger profile は $1000000$ joint states で closure せず、
   正しい結論は `INCONCLUSIVE` である。
   [profile 境界の記録](https://github.com/hora-algebra/exploring-math/blob/main/generalized-star-height/CORE2_SCHUETZENBERGER_PROFILE_COMPLETENESS_20260722.md)
   は、有限深さでの point separation から一様な fibre separation を
   推論できないことも unary counterexample で示す。従って
   Schützenberger profile の有界深さ十分性は、依然として CORE2 と
   同程度の theorem-strength obligation である。

### `gsh_bootstrap` への具体的な含意

- `GLOBAL-ONE` の文は変更しない。CORE2 はそれと同値な別インターフェース
  として追加すべきであり、解決済み補題として扱ってはならない。
- 最優先の Lean 移植候補は巨大な有限計算ではなく、まず scalar CORE2 の
  定義と上の双方向同値である（`L-CORE2-EQV-001`）。
- 4096族、$A_5$ raw stamp、`C4` profile の数値は、hash-pinned checker を
  本リポジトリで独立再生するまで外部 `UNREVIEWED` のままにする
  （`M-EXP-PR2-001`）。
- `a5_235`、全60元アルファベット版 $A_5$、フル版 $L_2$ の
  generalized star-height は PR #2 でも未解決であり、候補リストから
  除外しない。
